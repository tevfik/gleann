## Overview

This benchmark compares gleann's 4 document extraction backends across speed,
structural quality, and content fidelity using the same test documents.

### Extraction Layers

| Layer | Backend | Type | Dependencies | When Used |
|:-----:|---------|------|--------------|----------|
| -1 | **go-native** | In-process (Go) | Zero external deps | Always available (fallback) |
| 0 | **markitdown-cli** | CLI subprocess | Python `markitdown` | When markitdown is installed |
| 1a | **plugin-docs** | HTTP server | Python markitdown + docling | When plugin is installed |
| 1b | **plugin-marker** | HTTP server | Python marker-pdf + surya OCR | When plugin is installed |

### Fallback Chain

```
gleann index build → for each file:
  1. Try plugin (Layer 1a/1b) via HTTP /convert
  2. If no plugin → try markitdown CLI (Layer 0)
  3. If no markitdown → Go-native (Layer -1)
```

This means **gleann always works** even without any plugins installed.
Plugins add higher-quality extraction for specific formats.

## Analysis

### Speed

- **go-native** is the fastest backend since it runs in-process with zero network overhead.
- **markitdown-cli** adds Python subprocess startup cost (~300-500ms per file).
- **plugin-docs** amortizes startup via persistent HTTP server; docling PDF parsing adds ~500ms.
- **plugin-marker** is slowest due to deep learning model inference (surya OCR).

### Quality

- **go-native** extracts text but produces limited structural metadata (few or no headings from PDF).
- **markitdown-cli** produces markdown but often without heading markers for PDF content.
- **plugin-docs (docling)** produces the best PDF structure: deep section hierarchy, table detection.
- **plugin-marker** matches docling on section detection, adds OCR capability for scanned documents.

### Recommendations

| Use Case | Recommended Backend | Reason |
|----------|-------------------|--------|
| No dependencies available | **go-native** | Zero deps, always works |
| Lightweight install | **markitdown-cli** | `pip install markitdown`, no server |
| Production PDF workflow | **plugin-docs (docling)** | Best speed/quality balance for PDF |
| Scanned documents / OCR | **plugin-marker** | Only backend with deep-learning OCR |
| Office documents (DOCX/XLSX) | **plugin-docs** or **go-native** | Both handle Office formats well |
| HTML extraction | **go-native** or **plugin-marker** | Both support HTML natively |
| Maximum format coverage | **plugin-marker** | 15+ formats incl. EPUB, TIFF |
